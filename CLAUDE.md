# mobile_meme_bot

Телеграм-бот, который раз в день постит случайный IT-мем в топик группы kbtumobile.

## Архитектура

Нет долгоживущего процесса — это one-shot async скрипт, запускаемый по расписанию GitHub Actions. Состояние (какие мемы уже публиковали) хранится прямо в репозитории и коммитится обратно после каждого запуска.

Три связанных куска:

1. **`bot.py`** — параллельно ходит в `meme-api.com/gimme/{sub}/50` по всем сабам, фильтрует картинки, выбирает случайную среди невиданных, обновляет `state.json`, отправляет `send_photo` в топик. Падает с ненулевым кодом, если все запросы вернули пусто — тогда workflow не коммитит state.
2. **`state.json`** — плоский список md5-хэшей уже опубликованных постов. Коммитится ботом после каждого успешного поста.
3. **`.github/workflows/meme.yml`** — cron `0 10 * * *` UTC (13:00 Алматы) + `workflow_dispatch` для ручного теста. Ставит deps, прогоняет `bot.py` с `BOT_TOKEN` из секрета, затем коммитит `state.json` обратно в main.

## Источник мемов

`https://meme-api.com/gimme/{sub}/50` — публичный прокси над Reddit ([D3vd/Meme_Api](https://github.com/D3vd/Meme_Api)). Используем его, а не Reddit напрямую, потому что Reddit блокирует анонимные запросы с облачных IP (GitHub-runner'ы получают 403). Агрегатор ходит в Reddit со своими OAuth-креденшелами и кэширует ответы.

Ответ: `{"count": 50, "memes": [{"title", "url", "postLink", "subreddit", "nsfw", "spoiler", ...}]}`.

Риск: третья сторона. Если API упадёт — бот просто выйдет с кодом 1, workflow не закоммитит state, следующий запуск попробует снова.

## Схема state.json

```json
{"seen": ["<md5(post.url)[:10]>", "..."]}
```

Плоская, без тегов/категорий — один жанр контента (IT-мемы) в один топик. Хэшим `url` (а не `postLink`/id), потому что у meme-api в ответе нет Reddit post.id, а url однозначно идентифицирует картинку — если кто-то зарепостит тот же мем в другой саб, мы его всё равно пропустим.

Когда все кандидаты из текущей выдачи уже были показаны — `seen` чистится (reset cycle), логируется `cycle complete — seen reset`, и выбирается случайный мем из свежей выдачи заново.

## Как добавить/убрать subreddit

Редактируешь константу `SUBS` в `bot.py` — и всё. Никакой конфиг-файл не нужен.

```python
SUBS = [
    "ProgrammerHumor",
    "programmingmemes",
    ...
]
```

## Почему такие фильтры

meme-api сам отдаёт только картинки из `hot` (без стикнутых и видео), так что работы меньше, чем при прямом Reddit. Оставили:

- **`nsfw`** — NSFW-контент нельзя постить в публичный топик студенческой группы.
- **`spoiler`** — спойлер-посты обёрнуты в Reddit-preview, Telegram не покажет их как нормальную картинку.
- **Расширение `.jpg/.jpeg/.png/.gif/.gifv`** — страховка от того, что агрегатор пропустит не-картиночный URL (видели репорты про i.redd.it-видео иногда). `send_photo` по не-картиночному URL вернёт 400 — лучше отфильтровать заранее.

## Telegram constraints

- **Caption ≤ 1024 символов**. У нас caption = `title + \n\n + "r/{sub} · {postLink}"`. `postLink` — полный reddit-URL, может быть ~120 символов, поэтому title усекается до 900 — запас на остальное.
- **`message_thread_id=TOPIC_ID`** обязателен, иначе бот постит в General, а не в нужный топик.

## .gifv → .gif хак

Imgur отдаёт `.gifv` как HTML-страницу с `<video>`-тегом — Telegram не сможет её скачать по URL. Но если заменить `.gifv` на `.gif` (отрезать последнюю букву), тот же CDN отдаст полноценный gif-файл. Один символ кода → работают imgur-гифки.

## Секреты

`BOT_TOKEN` лежит только в GitHub Secrets (`Settings → Secrets and variables → Actions`). В коде — `os.getenv("BOT_TOKEN")`. Не логируем, в файлы не пишем.
